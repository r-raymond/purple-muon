loadAsset :: (MonadIO m, MonadError Text m) => al -> FilePath -> m [AssetID (Asset al)] Source #

Load an asset (maybe multiple assets) into an AssetLoader. Returns the AssetIDs of the loaded types.

WARNING: loadAsset will always normalize the given FilePath via the cabal paths package. Hence all paths should be specified relative to the root of the git repository. Also make sure that the files are mentioned in the package.yaml file under data-files.

data HashmapLoader a ext Source #

A simple implementation of an AssetLoader via Hashmaps a is the type of the Asset to be loaded and stored and ext can be any extra data that needs to be used when loading an asset from a file.
